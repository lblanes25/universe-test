# Stage 2 Handoff Review — Run Artifacts

Generated by `src/stage2_handoff_review/`. Mix of tracked and local-only:

**Tracked in git** — so the same batches can run from any clone:
- `_dry_run_summary.md` — batch composition + token estimates.
- `batches/batch_NNN/prompt.md` — paste-ready prompt for ChatGPT Pro.
- `batches/batch_NNN/manifest.json` — batch metadata (focal/target/source IDs, token estimates).

**Local-only** (in `.gitignore`) — confidential by design:
- `batches/batch_NNN/response.json` — you paste ChatGPT's response here.
- `aggregated/findings.csv`, `aggregated/gate_log.md`, `aggregated/ranked_summary.md` — derived from responses.

## Workflow per batch

1. Open `batches/batch_001/prompt.md` in an editor.
2. Copy the entire contents.
3. Paste into a fresh ChatGPT Pro conversation (extended thinking enabled).
4. Wait for the response. It should be one JSON object in a ```` ```json ... ``` ```` code block.
5. Copy the response (entire message, or just the code block — the aggregator handles both).
6. Save to `batches/batch_001/response.json` (overwrite if exists).
7. Run `python -m src.stage2_handoff_review.aggregate` from the repo root.
8. Check `aggregated/gate_log.md`. If batch 1 fails the gate, do not proceed; revise prompt or payload and re-run batch 1.
9. If the gate passes, repeat for the next batch.

## Re-running after changes

- `python -m src.stage2_handoff_review.generate --dry-run` — preview composition only.
- `python -m src.stage2_handoff_review.generate` — regenerate prompt files. Skips any batch that already has a non-empty `response.json` (resume-safe).
- `python -m src.stage2_handoff_review.aggregate` — re-aggregate. Safe to run repeatedly.
- `python -m src.stage2_handoff_review.aggregate --override-gate` — aggregate all batches even if batch 1 fails the gate (debugging only).

## Gate checks

Applied to **batch 1 in the processing order** (first batch with a response). If it fails, aggregation stops and prints which check failed.

1. Response is valid JSON matching the declared schema.
2. Findings are scoped to focal entities (no findings on target/source context).
3. At least one Task 5 finding references a handoff partner present in the batch.
4. Task 3 findings: ≥70% cite `evidence_layer: control`.
5. Task 5 findings: ≥50% cite `evidence_layer: control`.
6. Response includes a `ranked_summary` with all three keys.

Thresholds live in `config/stage2_batching.json` under `gate_thresholds`.
